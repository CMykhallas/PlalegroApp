```markdown
# Guia de Acessibilidade

## Princípios

- **Contraste**: cores com alto contraste para facilitar leitura.
- **Fonte legível**: uso de fontes sans-serif e tamanho mínimo de 16px.
- **Navegação por teclado**: todos os elementos interativos acessíveis via `Tab`.
- **Compatibilidade com leitores de tela**: uso de `aria-label` e semântica HTML correta.

## Boas práticas

- Evitar uso exclusivo de cores para transmitir informação.
- Garantir que botões tenham rótulos claros.
- Fornecer feedback auditivo/visual em interações.

## Ferramentas de validação

- Lighthouse (Google Chrome).
- axe DevTools.
- WAVE Accessibility Tool.
```
