# ADR-0005: Uso do Vercel para Deploy Contínuo do Next.js (Plalegro App)

## Status

Aceito ✅

## Contexto

O projeto **Plalegro App (Web)** é construído em **Next.js** e precisa de uma plataforma de deploy contínuo que:

- Seja otimizada para aplicações React/Next.js.
- Ofereça integração simples com GitHub Actions e pipelines CI/CD.
- Permita ambientes separados (staging e produção).
- Garanta escalabilidade e performance global.

Alternativas como Netlify, AWS Amplify e Google Cloud foram consideradas, mas a equipe já possui experiência com Vercel e o framework Next.js foi criado pela própria Vercel, garantindo suporte nativo.

## Decisão

Adotar **Vercel** como plataforma oficial de deploy contínuo para o Plalegro App:

- Deploy automático em cada push para `develop` (staging).
- Deploy automático em cada push para `main` (produção).
- Integração com GitHub Actions (`staging.yml` e `cd.yml`).
- Suporte nativo a **next/font**, **ISR (Incremental Static Regeneration)** e **Edge Functions**.
- Monitoramento de performance e logs centralizados.

## Consequências

- **Positivas:**
  - Deploy rápido e automatizado.
  - Suporte nativo ao Next.js.
  - Ambientes separados para QA e produção.
  - Escalabilidade global com CDN integrada.
- **Negativas:**
  - Dependência da infraestrutura Vercel.
  - Custos adicionais em planos pagos para features avançadas.
  - Menor flexibilidade comparado a soluções customizadas em cloud.

## Alternativas consideradas

- **Netlify:** boa opção, mas menos otimizada para Next.js.
- **AWS Amplify:** poderoso, mas mais complexo de configurar.
- **Google Cloud Run:** flexível, mas exige maior esforço de DevOps.

## Decisão final

O **Vercel** será usado como plataforma oficial de deploy contínuo para o Plalegro App.  
Será integrado aos pipelines GitHub Actions, garantindo deploy automático em staging e produção.
