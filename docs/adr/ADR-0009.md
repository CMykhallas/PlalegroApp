# ADR-0009: Deploy e Infraestrutura

## Status

Aceito ✅

## Contexto

O projeto precisa de deploy contínuo para:

- **Web (Next.js)** → Vercel.
- **Android (APK)** → Firebase App Distribution.
- **Backend (Ktor Server)** → ambiente staging/produção.

## Decisão

Adotar infraestrutura baseada em:

- **GitHub Actions** para CI/CD.
- **Vercel** para deploy web.
- **Firebase App Distribution** para Android.
- **Ktor Server** hospedado em ambiente cloud (Heroku/Fly.io/Kubernetes).

## Consequências

- **Positivas:** automação completa, ambientes separados, integração nativa.
- **Negativas:** dependência de múltiplos serviços externos.

## Alternativas consideradas

- Jenkins: rejeitado por exigir manutenção própria.
- GitLab CI: rejeitado por não ser o repositório principal.
- Deploy manual: rejeitado por falta de automação.

## Decisão final

Infraestrutura oficial: **GitHub Actions + Vercel + Firebase + Ktor Server**.
