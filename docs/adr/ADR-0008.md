# ADR-0008: Uso de Kotlinx Serialization para serialização de dados

## Status

Aceito ✅

## Contexto

O projeto precisa serializar entidades e DTOs para:

- Persistência local (JSON em assets).
- Comunicação via API (Ktor Server).
- Compatibilidade multiplataforma (Android, JS, JVM).

## Decisão

Adotar **Kotlinx Serialization**:

- Suporte nativo a multiplataforma.
- Integração com Ktor Client/Server.
- Configuração simples com `@Serializable`.

## Consequências

- **Positivas:** serialização consistente, multiplataforma, integrada ao ecossistema Kotlin.
- **Negativas:** dependência da lib, necessidade de anotações em modelos.

## Alternativas consideradas

- Gson/Moshi: boas opções, mas não multiplataforma.
- Jackson: poderoso, mas pesado e não multiplataforma.

## Decisão final

Usar **Kotlinx Serialization** como solução oficial de serialização.
