# ADR-0004: Uso do Firebase App Distribution para entrega de builds Android

## Status

Aceito ✅

## Contexto

O projeto **Play Learn Grow** precisa distribuir versões de teste do aplicativo Android para:

- **Equipe de QA** (validação em staging).
- **Testers internos e externos** (validação em produção antes de release público).

A distribuição manual de APKs é ineficiente, insegura e difícil de rastrear. Precisamos de uma solução que:

- Automatize a entrega de builds.
- Permita controle de grupos de testers.
- Integre com pipelines CI/CD.
- Ofereça métricas de uso e feedback.

## Decisão

Adotar **Firebase App Distribution** como plataforma oficial de entrega de builds Android:

- Integração direta com GitHub Actions.
- Upload automático de APKs gerados em pipelines (`staging.yml` e `cd.yml`).
- Gerenciamento de grupos de testers (QA, beta testers).
- Notificações automáticas para testers quando novos builds estão disponíveis.
- Dashboard centralizado para feedback e métricas.

## Consequências

- **Positivas:**
  - Distribuição rápida e segura de builds.
  - QA e testers recebem notificações automáticas.
  - Feedback centralizado no Firebase Console.
  - Integração transparente com CI/CD.
- **Negativas:**
  - Dependência da infraestrutura Firebase.
  - Necessidade de configurar tokens e credenciais seguras.
  - Testers precisam instalar o app Firebase App Tester.

## Alternativas consideradas

- **Google Play Internal Testing:** bom para releases próximos da produção, mas menos flexível para QA rápido.
- **AppCenter (Microsoft):** opção robusta, mas equipe já utiliza Firebase para outros serviços.
- **Distribuição manual via APK:** rejeitada por falta de automação e rastreabilidade.

## Decisão final

O **Firebase App Distribution** será usado como solução oficial para entrega de builds Android em QA e produção.  
Será integrado aos pipelines GitHub Actions (`staging.yml` e `cd.yml`), garantindo distribuição contínua e feedback rápido.
