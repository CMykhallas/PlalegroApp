# ADR-0010: Versionamento de Conteúdo Educativo

## Status

Aceito ✅

## Contexto

O aplicativo distribui pacotes de conteúdo educativo (`content_packs.json`).  
É necessário versionar esses pacotes para:

- Garantir consistência entre Android e Web.
- Permitir rollback em caso de erro.
- Suportar múltiplos idiomas e atualizações incrementais.

## Decisão

Adotar versionamento semântico (`major.minor.patch`) para pacotes de conteúdo:

- `major`: mudanças estruturais (ex.: novo formato).
- `minor`: novos conteúdos adicionados.
- `patch`: correções menores (ex.: textos, traduções).

Armazenamento:

- `app/src/main/assets/content_packs.json` (Android).
- Repositório remoto sincronizado para Web/Server.

## Consequências

- **Positivas:** controle de evolução, rollback seguro, consistência multiplataforma.
- **Negativas:** necessidade de disciplina na gestão de versões.

## Alternativas consideradas

- Versionamento manual sem padrão: rejeitado por risco de inconsistência.
- Apenas timestamp: rejeitado por falta de semântica.

## Decisão final

Usar **versionamento semântico** para pacotes de conteúdo educativo.
