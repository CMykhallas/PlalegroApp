# ADR-0007: Uso de Detekt + Ktlint para análise estática e formatação

## Status

Aceito ✅

## Contexto

O projeto é majoritariamente em Kotlin (Android + Shared KMP).  
Precisamos garantir:

- Qualidade de código consistente.
- Padronização de estilo.
- Detecção precoce de problemas (complexidade, código morto, más práticas).

## Decisão

Adotar **Detekt** para análise estática e **Ktlint** para formatação automática:

- Detekt: regras de complexidade, estilo, segurança.
- Ktlint: padronização de indentação, espaçamento, imports.
- Integração com CI (ci.yml) para bloquear merges com violações.

## Consequências

- **Positivas:** código limpo, consistente, fácil de manter.
- **Negativas:** curva inicial de configuração, necessidade de ajustes em código legado.

## Alternativas consideradas

- Apenas Ktlint: não cobre análise estática.
- Apenas Detekt: não garante formatação automática.
- Ferramentas customizadas: maior esforço de manutenção.

## Decisão final

Usar **Detekt + Ktlint** como ferramentas oficiais de qualidade de código Kotlin.
